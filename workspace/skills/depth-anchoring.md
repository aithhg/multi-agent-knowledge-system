# 深度锚定机制

> 输出前必须定义成功标准，确保输出有深度、有价值。

---

## 深度锚定三问

在开始任何输出之前，必须先回答这三个问题：

### 问题 1：目标读者是谁？

**定义**：这篇输出给谁看？

| 选项 | 说明 |
|------|------|
| 入门新手 | 对该领域完全不了解的人 |
| 进阶学习者 | 有基础但需要系统性提升 |
| 从业者 | 在该领域有实践经验 |
| 专家 | 需要深度洞察和前沿观点 |

**回答**：____

---

### 问题 2：读者获得什么？

**定义**：读者读完这篇内容后，能获得什么原文没有的价值？

| 价值类型 | 说明 |
|----------|------|
| 理解门槛降低 | 把复杂的讲简单 |
| 视角拓展 | 提供了原文没有的新角度 |
| 实践指导 | 给出了可操作的方法 |
| 认知刷新 | 挑战了已有的错误认知 |
| 知识关联 | 连接了碎片化的知识 |

**回答**：
- 主要价值：____
- 次要价值：____

---

### 问题 3：核心观点是什么？

**定义**：如果只能用一句话概括这篇文章/产出，是什么？

**回答**：____

---

## 深度锚定检查

完成输出后，回顾锚定三问：

- [ ] 输出的内容是否服务于「目标读者」？
- [ ] 是否真的交付了「读者获得」中承诺的价值？
- [ ] 「核心观点」是否在输出中得到了充分展开？

如果任何一个答案是否定的，**返回修改**。

---

## 使用场景

以下场景必须使用深度锚定机制：

| 场景 | 是否必须 |
|------|---------|
| 生成公众号长文 | ✅ 必须 |
| 生成研究备忘录 | ✅ 必须 |
| 生成摘要 | ✅ 必须 |
| 生成概念卡 | ✅ 必须 |
| 生成社交媒体文案 | 建议使用 |
| 回答知识性问题 | 建议使用 |

---

## 与质量门禁的关系

深度锚定 **先于** 质量门禁执行：

```
1. 回答深度锚定三问（定义成功标准）
     ↓
2. 执行输出
     ↓
3. 通过质量门禁清单检查
     ↓
4. 输出完成
```
