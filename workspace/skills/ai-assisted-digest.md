# AI 辅助切入点识别

> 上传 raw 文件后，AI 先分析内容结构，输出 3-5 个可消化方向供用户选择。

---

## 流程说明

```
用户：帮我消化这个视频
     ↓
AI：读取 raw 文件，预分析内容结构
     ↓
AI：输出 3-5 个可消化方向候选
     ↓
用户：选择方向 A
     ↓
AI：按方向 A 进行深度消化
```

---

## AI 预分析提示词模板

读取 raw 文件后，使用以下提示词框架：

```
## 预分析任务

分析以下内容的结构，识别 3-5 个可消化的方向：

### 内容概要
- 来源：[来源类型]
- 主题：[1句话主题]
- 时长/篇幅：[长度]

### 内容结构分析
[分析内容的组织方式：是否有时序/因果/对比/分类等结构]

### 可消化方向候选

**方向A：[方向名称]**
- 适用性：[为什么适合从这个角度消化]
- 核心产出：[会产出什么]
- 目标读者：[谁最需要这个]

**方向B：[方向名称]**
- ...

**方向C：[方向名称]**
- ...

### 推荐优先级
综合考虑内容的完整性、可操作性、价值密度，推荐：
1. 首选：[方向X]
2. 次选：[方向Y]
```

---

## 方向类型参考

| 方向类型 | 适用场景 | 产出 |
|---------|---------|------|
| **方法论框架** | 内容有系统性方法/步骤/流程 | 可复用的方法论模板 |
| **案例分析** | 内容以案例/故事为主 | 案例卡片 + 启发点 |
| **概念澄清** | 内容澄清一个/多个概念 | 概念卡 + 边界条件 |
| **工具资源** | 内容推荐了工具/资源 | 资源清单 + 使用场景 |
| **反常识点** | 内容颠覆常规认知 | 反常识卡片 + 适用边界 |
| **数据洞察** | 内容包含关键数据 | 数据卡片 + 可视化建议 |

## 使用示例

**输入**：
```
用户：帮我消化这个 B站视频
文件：raw/20260428-bilibili-ai-agent-trend.md
```

**AI 输出**：
```
## 可消化方向候选

### 内容概要
- 来源：B站视频（45分钟）
- 主题：AI Agent 发展趋势与落地场景
- 时长：45分钟，含多个访谈片段

### 可消化方向候选

**方向A：AI Agent 方法论**
- 适用性：视频系统介绍了 Agent 的设计模式
- 核心产出：Agent 设计模式概念卡
- 目标读者：想了解 Agent 技术的开发者

**方向B：行业案例集**
- 适用性：包含多个不同行业的落地案例
- 核心产出：案例卡片 × 5
- 目标读者：关注 AI 落地的人

**方向C：创业者访谈精华**
- 适用性：多位创业者的观点集合
- 核心产出：观点摘录 + 洞察
- 目标读者：AI 创业者

### 推荐优先级
1. 首选：方向A（方法论最完整）
2. 次选：方向B（案例最丰富）

请选择消化方向：[A / B / C]
```

## 与消化启动模板的关系

- **消化启动模板**（video-template.md / article-template.md）：用户手动填写，找到切入点
- **AI 辅助切入点识别**：AI 自动分析，提供方向候选，两者可叠加使用

建议：先用 AI 辅助识别方向，再用消化启动模板细化执行。
