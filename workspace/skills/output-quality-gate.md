# 输出质量门禁清单

> 所有输出流程中，输出前必须通过以下检查。

---

## 质量门禁检查表

### 事实性检查

- [ ] **所有事实/数据有原始来源吗？**
  - 如果有，标注来源：`来源：[[raw文件id]]`
  - 如果无法确认，标注：`[待验证]`

- [ ] **没有捏造人名/地名/数字吗？**
  - 禁止凭空编造具体数字（"研究表明..."）
  - 禁止编造引用（"XX曾说..."）

- [ ] **内部一致性检查**
  - 标题和内容是否匹配？
  - 结论是否由论据支撑？

### 深度检查

- [ ] **有没有比原文更深的洞察？**
  - 不只是复述原文内容
  - 有作者自己的分析和综合

- [ ] **有没有建立与现有知识的关联？**
  - [[已有概念卡 1]]
  - [[已有概念卡 2]]

- [ ] **读者能从中获得原文没有的什么价值？**
  - 理解门槛降低了？
  - 有新的视角？
  - 有实践指导？

### 格式检查

- [ ] **是否使用了对应输出类型的固定模板？**
  - 摘要 → `summary` Schema
  - 概念卡 → `concept` Schema
  - Memo → `memo` Schema
  - 文章 → `khazix-writer` 风格

- [ ] **标题/结构/长度是否符合要求？**

### 禁止事项

以下情况必须修正后才能输出：

| 禁止项 | 示例 | 修正方式 |
|--------|------|---------|
| 模糊量化 | "很多人说"、"研究表明" | 提供具体数字或标注「多数/少数」 |
| 绝对化表述 | "永远不"、"所有人" | 改为「在某些情况下」「大多数人」 |
| 堆砌结构 | 超过3层小标题 | 合并或删除中间层级 |
| 空洞金句 | 无实质内容的鸡汤 | 删除或补充实质内容 |

---

## 通过标准

**所有 Must Have 项必须全部通过才能输出。**

| 检查项 | 必须通过 |
|--------|---------|
| 事实有来源/待验证标注 | ✅ |
| 无捏造 | ✅ |
| 有深度增量 | ✅ |
| 格式符合模板 | ✅ |
